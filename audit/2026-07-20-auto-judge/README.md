# 自動工程判定の実データ監査 (2026-07-20)

Phase A / A.1 で「自動工程かどうか」の判定を `src/domain/workExecution.js` の共通関数へ一本化した際の、
**本番実データによる影響調査**の記録です。第三者が同じ結果を再現できるように、使ったスクリプトと結果JSONをそのまま残します。

## ⚠ 実行スクリプトはこのリポジトリに置きません（2026-07-20 削除）

調査に使った `.mjs` は、**匿名認証で本番Firestoreを全件読み出す動くスクリプト**でした。
このリポジトリは公開されているため、「そのまま実行できる本番の読み出し口」を配ることになります。
現行のセキュリティルールが `allow read, write: if request.auth != null`（匿名認証でも通る）である以上、
公開しておくべきものではないと判断し、**結果のJSONだけを残して削除しました**。

スクリプトはローカル（開発者の手元）に保管しています。再実行が必要な場合は開発担当へ依頼してください。

> 補足（正直に書きます）: スクリプトを消しても、**アプリ本体のソースが公開されている以上、
> 同等のものは書けます**。これは「入口の見つけやすさ」を下げただけで、根本対策ではありません。
> 根本対策はセキュリティルール側（名前空間の明示・権限の分離）です。

### 🎯金額・機械時間への影響

| 指標 | 変更前 | 変更後 | 差 |
|---|---|---|---|
| 候補在庫（金額） | ¥1,013,899 | ¥1,013,899 | **±0** |
| 機械時間（年） | 698 h | **601 h** | −97 h |

金額が動かないのは、拘束率 `autoLaborPct` が現在 **100%**（＝自動工程の時間も全額を人件費に入れている上限設定）だからです。
拘束率を実態値へ下げた時点で、判定の違いが金額に効いてきます。

機械時間が 698h → 601h に減ったのは、案②（マスタ参照）により
「三次元測定開始」の旧ロット（`executionMode` 未設定で保存されていたもの）が、
現行マスタの明示設定 `manual` に揃って手動扱いになったためです。
旧判定では名称の「測定開始」だけを見て自動としていました。

この 601h / ¥1,013,899 は、アプリ画面の表示と一致することを確認済みです。
